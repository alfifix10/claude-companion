Act as a senior visual designer paired with a frontend engineer. Read
`VISUAL_DESIGN_REVIEW.md` at the repo root and apply its 9-category
framework.

Focus areas for THIS project:
- Color hierarchy (is --accent overused? any tiering?)
- Typography scale (how many sizes/weights? purposeful?)
- Spacing consistency (does it follow a 4px / 8px grid?)
- Iconography (single style? consistent stroke width?)
- Motion quality (prefers-reduced-motion? easing curves?)
- CSS health (specificity, z-index system, duplicated keyframes)
- Dark-mode quality (pure black? pure white text? shadows muted?)
- Overlay patterns (history / settings / notice — cohesive?)

Always cite file + line. Propose design tokens where you'd solidify
scattered literal values. 3 "wow" findings + prioritised fix list.
